# 文章分享功能实现计划

## 概述

新增公开的文章分享接口，允许未登录用户通过分享链接访问设置为共享状态的文章。支持有密码和无密码两种分享模式。

## 需求总结

- 用户 A 通过现有的**更新文章接口**设置 `is_shared=True` 和可选的 `share_password`
- 新增**公开接口** `POST /api/share/<article_id>`，不需要登录即可访问
- 如果文章设置了密码，需要验证密码；无密码则直接返回内容
- 只有 `is_shared=True` 且未删除的文章才能被访问

# 分享文章内容模型（不包含敏感字段如 share_password、category_id）
share_article_model = article_ns.model('ShareArticle', {

### 步骤 3: 创建分享资源类

**文件**: `app/controllers/article/article_manager.py`

新增 `ShareArticleResource` 类：

```python
class ShareArticleResource(Resource):
    # 注意：不使用 @jwt_required() - 这是公开接口
    @article_ns.doc(description='访问分享文章')
    @article_ns.marshal_with(share_article_response_model)
    def post(self, article_id):
        """
        访问分享文章
        :params: password (可选，仅当文章设置了密码时需要)
        """
        try:
            # 1. 查询文章
            article = Article.get_article_by_id(article_id)
            
            # 2. 验证文章存在且未删除
            if not article or article.is_deleted:
                return {'code': 404, 'message': ARTICLE_ERROR_MESSAGE['NOT_FOUND']}, 404
            
            # 3. 验证文章已设置为分享状态
            if not article.is_shared:
                return {'code': 403, 'message': ARTICLE_ERROR_MESSAGE['NOT_SHARED']}, 403
            
            # 4. 密码验证（如果设置了密码）
            if article.share_password:
                parser = reqparse.RequestParser()
                parser.add_argument('password', type=str, required=False)
                args = parser.parse_args()
                
                if not args.get('password'):
                    return {'code': 400, 'message': ARTICLE_ERROR_MESSAGE['SHARE_PASSWORD_REQUIRED']}, 400
                
                if not verify_password_with_salt(article.share_password, args['password']):
                    return {'code': 401, 'message': ARTICLE_ERROR_MESSAGE['SHARE_PASSWORD_INCORRECT']}, 401
            
            # 5. 返回文章内容
            return {
                'code': 200,
                'message': ARTICLE_SUCCESS_MESSAGE['SHARE_ACCESS_SUCCESS'],
                'data': article
            }, 200
            
        except Exception as e:
            return {'code': 500, 'message': ARTICLE_ERROR_MESSAGE['COMMON_ERROR'] + ': ' + str(e)}, 500
```

## API 接口规范

### POST /api/share/<article_id>

**请求示例（无密码）**:
```http
POST /api/share/123e4567-e89b-12d3-a456-426614174000
Content-Type: application/x-www-form-urlencoded
```

**请求示例（有密码）**:
```http
POST /api/share/123e4567-e89b-12d3-a456-426614174000
Content-Type: application/x-www-form-urlencoded

password=mypassword123
```

**成功响应 (200)**:
```json
{
  "code": 200,
  "message": "分享文章访问成功",
  "data": {
    "id": "123e4567-e89b-12d3-a456-426614174000",
    "title": "我的笔记",
    "content": "这是文章的完整内容...",
    "type": "NOTE",
    "create_time": "2026-01-15 10:30:00",
    "update_time": "2026-02-20 15:45:00"
  }
}
```

**错误响应**:
| 状态码 | 说明 |
|--------|------|
| 400 | 需要提供密码但未提供 |
| 401 | 密码错误 |
| 403 | 文章未设置为分享状态 |
| 404 | 文章不存在或已删除 |
| 500 | 系统错误 |

## 验证测试

### 测试场景

1. **无密码分享文章**
   - 设置文章 `is_shared=True`，`share_password` 为空
   - POST 请求不带 password 参数，应返回 200 和文章内容

2. **有密码分享文章 - 正确密码**
   - 设置文章 `is_shared=True`，`share_password` 设置密码
   - POST 请求带正确 password，应返回 200 和文章内容

3. **有密码分享文章 - 错误密码**
   - POST 请求带错误 password，应返回 401

4. **有密码分享文章 - 未提供密码**
   - POST 请求不带 password，应返回 400

5. **访问未分享文章**
   - 访问 `is_shared=False` 的文章，应返回 403

6. **访问不存在/已删除文章**
   - 应返回 404

### 测试命令

启动服务后，可以使用 curl 测试：

```bash
# 测试无密码分享
curl -X POST http://localhost:5000/api/share/<article_id>

# 测试有密码分享
curl -X POST http://localhost:5000/api/share/<article_id> \
  -d "password=yourpassword"
```
