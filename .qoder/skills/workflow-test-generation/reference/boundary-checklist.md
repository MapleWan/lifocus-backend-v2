# Flask 项目边界条件 Checklist

针对 Flask + SQLAlchemy + RESTful API 项目的系统化边界条件检查清单。

---

## 1. API 请求参数

### 1.1 必填参数

| 边界 | 测试场景 | 示例 |
|------|----------|------|
| **参数缺失** | 不传必填参数 | `POST /login` 不传 `username` |
| **参数为 None** | 显式传 `null` | `{"username": null}` |
| **空字符串** | 传 `""` | `{"username": ""}` |
| **空白字符** | 传 `"   "` | `{"username": "   "}` |

### 1.2 字符串参数

| 边界 | 测试场景 | 示例 |
|------|----------|------|
| **最小长度** | 1 个字符 | `{"username": "a"}` |
| **最大长度** | 刚好等于限制 | `{"username": "a" * 64}` |
| **超长字符串** | 超过数据库字段限制 | `{"username": "a" * 100}` |
| **特殊字符** | Unicode、emoji | `{"username": "用户🔥"}` |
| **SQL 注入** | 恶意 SQL 字符 | `{"username": "' OR '1'='1"}` |
| **XSS 注入** | 脚本标签 | `{"name": "<script>alert(1)</script>"}` |

### 1.3 数值参数

| 边界 | 测试场景 | 示例 |
|------|----------|------|
| **零值** | `0` | `{"page": 0}` |
| **负数** | `-1` | `{"page": -1}` |
| **最大值** | `sys.maxsize` | `{"limit": 9223372036854775807}` |
| **浮点数** | 传入小数 | `{"count": 1.5}` |
| **非数字字符串** | `"abc"` | `{"page": "abc"}` |

### 1.4 布尔参数

| 边界 | 测试场景 | 示例 |
|------|----------|------|
| **true 变体** | `"true"`, `"1"`, `1`, `true` | `{"is_active": "true"}` |
| **false 变体** | `"false"`, `"0"`, `0`, `false` | `{"is_active": 0}` |
| **非法值** | `"yes"`, `"no"` | `{"is_active": "yes"}` |

### 1.5 日期/时间参数

| 边界 | 测试场景 | 示例 |
|------|----------|------|
| **ISO 8601 格式** | 标准格式 | `"2024-01-01T00:00:00Z"` |
| **非标准格式** | 其他格式 | `"2024/01/01"`, `"01-01-2024"` |
| **无效日期** | 不存在日期 | `"2024-02-30"` |
| **时区边界** | UTC vs 本地时间 | `"2024-01-01T00:00:00+08:00"` |
| **过去时间** | 已过期 | `"1970-01-01T00:00:00Z"` |
| **未来时间** | 远未来 | `"2099-12-31T23:59:59Z"` |

## 2. HTTP 层面

| 边界 | 测试场景 | 检查点 |
|------|----------|--------|
| **Content-Type** | 缺失或错误 | `application/json` vs `text/plain` |
| **请求体为空** | `POST/PUT` 不传 body | 返回 400 |
| **请求体过大** | 超大 JSON | 413 Payload Too Large |
| **URL 长度** | 超长 query string | 414 URI Too Long |
| **Header 注入** | `\n` 在 header 值中 | 安全检查 |
| **Method 不支持** | `PATCH`, `OPTIONS` | 返回 405 |

---

## 3. 认证/授权

| 边界 | 测试场景 | 检查点 |
|------|----------|--------|
| **Token 缺失** | 无 `Authorization` | 返回 401 |
| **Token 格式错误** | 非 `Bearer xxx` 格式 | 返回 401 |
| **Token 过期** | 使用过期 JWT | 返回 401 |
| **Token 伪造** | 篡改签名 | 返回 401 |
| **Token 刷新** | 用 access_token 调刷新接口 | 返回 401 |
| **权限不足** | 普通用户访问管理员接口 | 返回 403 |
| **资源越权** | 访问他人资源 | 返回 403 |

---

## 4. SQLAlchemy 模型

### 4.1 字段约束

| 边界 | 测试场景 | 检查点 |
|------|----------|--------|
| **nullable=False** | 字段传 `None` | 触发 `IntegrityError` |
| **unique=True** | 插入重复值 | 触发 `IntegrityError` |
| **String(64)** | 超过长度限制 | 数据截断或报错 |
| **default 值** | 不传该字段 | 检查默认值生效 |
| **onupdate** | 更新记录 | 检查更新时间自动更新 |

### 4.2 关系字段

| 边界 | 测试场景 | 检查点 |
|------|----------|--------|
| **外键不存在** | 关联 ID 不存在 | `ForeignKeyViolation` |
| **级联删除** | 删除父记录 | 检查子记录是否级联 |
| **懒加载** | 访问未加载关系 | `DetachedInstanceError` |
| **循环引用** | A→B→A | 序列化死循环 |

### 4.3 查询操作

| 边界 | 测试场景 | 检查点 |
|------|----------|--------|
| **空结果** | `.first()` 无记录 | 返回 `None` |
| **多条结果** | `.one()` 返回多条 | 抛出 `MultipleResultsFound` |
| **全表查询** | 无过滤条件的 `.all()` | 性能风险 |
| **分页越界** | page > 总页数 | 返回空列表 |
| **limit=0** | 分页大小为 0 | 返回空列表 |

## 5. 业务逻辑

### 5.1 用户认证

| 边界 | 测试场景 | 检查点 |
|------|----------|--------|
| **用户名不存在** | 登录时用户不存在 | 返回统一错误信息 |
| **密码错误** | 密码不匹配 | 返回统一错误信息 |
| **用户被禁用** | 登录已禁用账户 | 返回特定错误码 |
| **并发登录** | 同一账号多设备登录 | Token 独立有效 |

### 5.2 CRUD 操作

| 边界 | 测试场景 | 检查点 |
|------|----------|--------|
| **创建重复** | 唯一字段重复 | 返回 409 Conflict |
| **更新不存在** | 更新已删除记录 | 返回 404 |
| **删除不存在** | 删除不存在的 ID | 返回 404 |
| **删除已删除** | 软删除后再删除 | 返回 404 |
| **批量操作** | 空 ID 列表 | 返回空结果或报错 |

### 5.3 软删除

| 边界 | 测试场景 | 检查点 |
|------|----------|--------|
| **查询包含已删除** | 默认过滤条件 | 不返回已删除记录 |
| **恢复已删除** | 恢复软删除记录 | `is_deleted` 重置 |
| **重复删除** | 已删除再次删除 | 返回 404 |

## 6. 文件上传

| 边界 | 测试场景 | 检查点 |
|------|----------|--------|
| **空文件** | 0 字节文件 | 拒绝或允许 |
| **超大文件** | 超过限制 | 返回 413 |
| **非法格式** | 非允许 MIME 类型 | 返回 400 |
| **伪造扩展名** | `.jpg` 实际是 `.exe` | 内容检查 |
| **文件名特殊字符** | `../../../etc/passwd` | 路径遍历防护 |
| **同名文件** | 上传重复文件名 | 重命名或覆盖策略 |

---

## 7. 响应处理

| 边界 | 测试场景 | 检查点 |
|------|----------|--------|
| **空列表** | 查询无结果 | `{"data": [], "total": 0}` |
| **大数据集** | 返回大量记录 | 分页处理 |
| **循环引用** | 模型自关联 | JSON 序列化错误 |
| **Decimal 类型** | SQLAlchemy Decimal | JSON 可序列化 |
| **datetime 类型** | 时区处理 | 统一 UTC 或本地时间 |
| **None 值** | 字段为 `None` | 返回 `null` 而非省略 |

---

## 8. 异常处理

| 边界 | 测试场景 | 预期行为 |
|------|----------|----------|
| **数据库连接失败** | MySQL 不可用 | 返回 500，不暴露详情 |
| **事务回滚** | 中间步骤失败 | 数据一致性 |
| **超时** | 慢查询 | 返回 504 |
| **并发写入** | 乐观锁冲突 | 重试或返回 409 |

## 使用方法

根据被测接口/函数的特征，选择适用的边界条件：

**示例 1：用户登录接口 `POST /login`**
```python
# 必填参数边界
- username: 缺失、空字符串、超长、SQL 注入
- password: 缺失、空字符串

# 业务边界
- 用户名不存在
- 密码错误
- 用户被禁用
```

**示例 2：创建文章接口 `POST /articles`**
```python
# 参数边界
- title: 空字符串、超长(>255)、None
- content: 空内容、超大文本
- category_id: 不存在的外键

# 认证边界
- Token 缺失、过期、伪造
- 无权限用户

# 数据库边界
- 标题唯一性冲突
```

**示例 3：SQLAlchemy 模型方法**
```python
# User.get_user_by_username()
- username: None、空字符串、不存在用户名
- 返回: 存在返回 User，不存在返回 None

# User.add_user()
- 必填字段缺失: username=None 触发 IntegrityError
- 唯一字段重复: 重复 username 触发 IntegrityError
```
